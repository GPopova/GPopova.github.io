Тестовое задание (фронтенд-разработчик)
В рамках выполнения тестового задания на свое усмотрение оформила внешний вид страницы. Использовала следующие инструменты: -Bootstrap(grid.css); -плагин Owl Carousel; -gulp(для сборки); -плагин jQuery Templates(для генерации html-элементов из объектов данных JavaScript). html,css, javascript,jQuery,ajax. 
Я не владею знаниями фрейморков.js, поэтому обходилась без них. Т.к. сервис: http://smktesting.herokuapp.com/ запрещает кроссдоменные запросы, в качестве бэк-энда воспользовалась www.mockable.io.
Выполнение требований(касающихся работы с сервером) в структуре проекта: 
1. В шапке страницы "Sign in" открывает форму входа/регистрации для пользователя. После успешного заполнения формы(присутствует валидация), отправляет имя и пароль пользователя на сервер, в ответ получает статус и token. После их получения пользователю становится доступна возможность оставлять комментарии.
2. Сами товары находятся в "теле" страницы с заголовком "Best choise" (class="products"), они подгружаются с сервера единажды при загрузке страницы. Комментарии пользователей подгружаются динамически единажды к каждому продукту при нажатии "reviews" и "прячутся" так же.
3. Для того, чтобы оставить комментарий достаточно его ввести и отправить(тут проверки на заполнение отсутствуют). 
4. Поскольку продукта всего 2, смысла в скрытии какого-то от всех пользователей не вижу.
